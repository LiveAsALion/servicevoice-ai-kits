# Daily Note — 2026-04-01

## SpeedShield — Active Dev Session

Long debugging and feature session on SpeedShield (Flutter Android app).

### Bugs Found & Fixed
1. **BLUETOOTH_CONNECT not requested at runtime** — manifest had it declared but `main.dart` never called `Permission.bluetoothConnect.request()`. On Android 12+, `device?.name` returned null silently, breaking all BT detection. Fixed in `main.dart`.

2. **Native receivers never saw activationMode or vehicleBluetoothDevices** — `SettingsService` was saving a single JSON blob under `alert_settings` key. Kotlin `BluetoothReceiver` was reading `flutter.activationMode` and `flutter.vehicleBluetoothDevices` — keys that were never written. Fixed: `SettingsService.save()` now writes flat keys alongside the blob.

3. **Startup BT state missing MAC address** — `SpeedShieldApp.kt` wrote name but not address on startup. If already connected at launch, MAC-based matching failed. Fixed to write both, and start monitoring immediately if device is in approved list.

4. **Double TTS alerts** — `MapScreen` and `background_service` each had their own `ProximityService` instance with independent alert state. Both fired TTS simultaneously when app was open. Fixed with `kAppForegroundKey` SharedPreferences flag — `MapScreen` writes true/false on lifecycle changes, background service skips TTS when flag is true.

5. **TTS announced wrong distance** — said configured threshold (e.g. "600 meters") instead of actual current distance. Fixed in `AlertService` — now takes `actualDistanceMeters` optional param, rounds to nearest 50m (primary) or 25m (secondary).

6. **`isForegroundMode()` doesn't exist** — used a non-existent method on `AndroidServiceInstance`. Replaced with the SharedPreferences flag approach.

7. **Unused import warning** — `flutter_background_service_android` import in `background_service.dart` was unnecessary after refactor.

8. **Deprecated Flutter APIs** — `activeColor`, `groupValue`, `withOpacity()` in `settings_screen.dart`. Replaced with `WidgetStateProperty`, `withValues(alpha:)`.

9. **Lint warnings** — bare `if` statements without braces, switch case formatting in `map_screen.dart`.

10. **`_prefsName` unused field** in `vehicle_detection_service.dart`. Removed.

### Key Lessons / Decisions (2026-04-01)
- Android runtime permissions are not optional — manifest declaration gets nothing on modern Android without the explicit Dart request
- BT matching should always use MAC, not name — names are user-editable and collide; MACs are hardware-stable
- SharedPreferences key contracts must match exactly between Dart writer and Kotlin reader — one blob written, multiple flat keys expected = silent failure
- Double-TTS is a foreground flag problem, not a deduplication problem — fix the trigger, not the output
- Multi-vehicle support requires a list model, not a string field; store both name and MAC for UX and matching respectively
