### Crash #1 — Android 14 foreground service notification
- **Error:** `RemoteServiceException$CannotPostForegroundServiceNotificationException`
- **Cause:** Android 13+ requires `POST_NOTIFICATIONS` permission for foreground service
- **Fix:** Added `POST_NOTIFICATIONS` to AndroidManifest.xml + runtime `Permission.notification.request()` in `main.dart`
- Result: App launched successfully after fix, no crash

### Crash #2 — BootReceiver ClassNotFoundException
- **Error:** `ClassNotFoundException: com.speedshield.app.BootReceiver`
- **Cause:** BootReceiver declared in AndroidManifest.xml but no Kotlin class ever created
- **Fix:** Removed BootReceiver declaration and RECEIVE_BOOT_COMPLETED permission from manifest (deferred to v2)
- Note: This crash was present from the start but masked by crash #1

### Key Lessons / Decisions
- Android 14 foreground services require POST_NOTIFICATIONS runtime permission — not just manifest entry
- Never declare a receiver in AndroidManifest without the backing Kotlin class
- Notification channels must be created in Application.onCreate(), not MainActivity
- autoStart on background service can race against permission dialogs — set false, start manually post-permission
- FlutterBluePlus async calls in widget _load() can silently crash the entire init flow — keep them out
